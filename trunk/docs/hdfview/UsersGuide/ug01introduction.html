1.0 What It Is

The HDFView is a Java-based tool for browsing and editing HDF4 and HDF5 files. HDFView allows users to browse through any HDF4 and HDF5 file; starting with a tree view of all top-level objects in an HDF file's hierarchy. HDFView allows a user to descend through the hierarchy and navigate among the file's data objects. The content of a data object is loaded only when the object is selected, providing interactive and efficient access to HDF4 and HDF5 files. HDFView editing features allow a user to create, delete, and modify the value of HDF objects and attributes.

The HDFView GUI is simple and easy-to-use. First, HDFView was implemented by using JavaTM 2 Platform that is machine-independent. The GUI components have the same look-and-feel for all machines. Second, HDFView uses conventional folders and icons to display groups and datasets in a tree structure. Users can easily expand or collapse folders to navigate the hierarchical structure of an HDF file. Third, HDFView shows data content as text (table or plain text) or as image.

1.1 About This Release

This release includes bug fixes, new features, test suite, and improved documentation.

HDF libraries

This release was built and tested with HDF4.2r5 and HDF5-1.8.2 with HDF5 1.6 compatibility flag. For information on New Features in HDF5 Release 1.8.0 and Format Compatibility Considerations, please visit http://www.hdfgroup.org/HDF5/doc/ADGuide/CompatFormat180.html.

  • Mac PowerPC
  • Mac Intel
  • Windows (Vista/XP/2000)
64-bit Java 2 SDK
  • Linux 64-bit AMD

New features

There are a lot of new features added to the release. The following is a list of major new features.
  • Support access to remote HDF5 files stored at iRODS server (Windows and Linux only). For information on HDF5-iRODS module, please visit http://www.hdfgroup.org/projects/irods/index.html
  • Shows large dataset (e.g. 1,000,000 x 100) very fast in table viewer.
  • Write variable length string data.
  • Support drag-and-drop files to HDFView.
  • Filter out fill value from image.
  • Import and apply palette from an external file.

1.2 Features

This version fixed bugs from previous versions and added a few new and enhanced features. It is a continuous work from previous HDFView. Some of the major features from previous versions include replaceable I/O and GUI modules, memory leak fixes, enhanced documentation, and test suite.

This version is user-configurable to support HDF4 or HDF5 or both. Users can choose HDFView features to support HDF4 or HDF5 or both when installing HDFView or compiling from the source code.

The following is a list of the browsing and editing features of HDFView. Details of the features are discussed in their related chapters.

1.2.1 Browsing Features

  • Plugable GUI modules
    • Tree Viewer
    • Table Viewer
    • Image Viewer
    • Text Viewer
    • Metadata Viewer
    • Palette Viewer
  • Supported file formats
    • HDF4
    • HDF5
    • netCDF (read only)
    • FITS (read only)
  • Tree view of file structure
    • Display file hierarchy in a tree and allow users to navigate among the file's data objects.
    • Add options to show dataset as an image, a table or text.
    • Allow users to select a subset of dataset to display.
    • Show attributes and other general information of data objects.
    • Support large Fonts in GUI components
    • Add -geometry switch to set the window size and location
  • Table view of file content
    • Display simple compound dataset (simple or nested) in table.
    • Allow users to selection select fields/members of VData/compound dataset to display.
    • Show multi-dimension compound dataset.
    • Display line plot for selected table data.
    • View variable-length datasets
    • View ENUM dataset
  • Image view of file content
    • Support simple image manipulation: zooming image in/out, flipping image horizontally or vertically.
    • Change image brightness
    • Animation display of 3D images.
    • Display histogram for selected image area.
    • Modify palette for index RGB images.
    • Support multiple color tables for HDF5 images.
    • Slider bar for modifying image threshold range
    • Filter “fill value" in Image Viewer
    • Show actual palette that results from palette manipulation
    • Support large number of 8kx8k 3D images by reuse the memory data buffer
    • Mouse Grab/Move to browse an image
  • Text view of file content
    • Display HDF5 user block in text, octal, hex and etc.
    • Option to show char dataset as text.
    • Show text attributes and text dataset in selected fonts (type and size).
    • show file information, such as file size, total number of datasets.

1.2.2 Editing Features

  • Add/delete/rename datasets and groups.
  • Edit content of datasets.
  • Copy/paste object in the same file or dross files of the same file format.
  • Export dataset values to ASCII text file.
  • Save an HDF4/HDF5 image to JPEG file.
  • Add/delete/modify attributes.
  • Convert JPEG image file into HDF4/HDF5 file.
  • Import data values from text file.
  • Add and modify user blocks.

1.3 Limitations / Known Problems

The current implementation of HDFView has the following limitations.
  • HDFView does not support undo operation; all the changes will go to the physical file, and the editing results cannot be recovered.
  • Deleting object/attribute in HDF4 is not supported.
  • HDFView does not support conversion between HDF4 and HDF5.
  • HDFView supports only indexed image with RGB color model or 24-bit true color image. It does not support other color models such as CMYK and HSV.
  • HDFView cannot modify compressed HDF4 SDS if rank > 2. When SDS has more than 2 dimensions, HDFView reads and displays only a hyperslab (say first plane). If user tries to modify a value and save the file, new data is not saved because HDF4 cannot do partial I/O on a compressed datasets. It can only overwrite the whole dataset

1.4 System Requirements

The HDFView is distributed in two forms: No_VM_bundled and VM_bundled. The No_VM_bundled distribution does not contain the Java Runtime Environment (JRE) while the VM_bundled distribution bundles the JRE.

To run HDFView with No_VM_bundled distribution, JDK1.5+ is required. The HDFView does not compile on jdk1.4.x or earlier version.

Download JavaTM 2 Platform at:

  • JavaTM 2 Platform, Standard Edition (Windows, Solaris, Linux)

The HDFView uses the HDF4 and HDF5 Java Native Interfaces, which calls the HDF4 and HDF5 library. These native interfaces are included in the distribution. For more information (source and pre-built) on the HDF4/5 Java Native Interface, visit website at /hdf-java-html/JNI/

Java Virtual Machine Required

The NoVM_bundled binary distribution does not contain the Java 2 Runtime Environment. The NoVM_bundled requires JDK1.5.0 or above. Before you install the NoVM_bundled distribution, you must have JDK1.5.0 or above installed on your machine. If you don't have a Java 2 VM, you must download and install one. As all releases of Macintosh OS X include a fully functional Java 2 Virtual Machine, there is no need for a Mac OS X JRE.

Download JavaTM 2 Platform at:
  • JavaTM 2 Platform, Standard Edition (Windows, Solaris, Linux)

StandAlone Application [JRE Bundled]

JRE1.5.0 is bundled in the VM_bundled distribution. You do not need any aditional software to run HDFView.

Download and installation of HDFView requires acceptance of the Java 2 Runtime Environment license agreement. Please read the Java 2 Runtime Environment Binary Code License Agreement before you download the VM_bundled distribution. The installation program will not install HDFView unless you accept the Java 2 Runtime Environment License Agreement.

Install on Unix or MAC OS X

After downloading the installation file, cd to the directory where you downloaded the installer. At the prompt type:
sh ./hdfview-xxx.bin,
where xxx is the Unix OS name such as solaris, linux or macosx. The installer will guide you to select the Java Virtual Machine and HDFView home directory (where you want to install the HDFView).

Install on Windows

After downloading, double-click
hdfview-windows.exe.

The installer will guide you to select the Java Virtual Machine and HDFView home directory (where you want to install the HDFView). After installing, you can launch the HDFView from
Start -> Program Files -> NCSA HDFView
